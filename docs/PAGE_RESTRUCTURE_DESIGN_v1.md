## 1. 배경 & 문제

현재 8개 페이지로 구성되어 있으나, 사용자 검토 결과 **3그룹**으로 중복 발견:

| 그룹 | 페이지 | 공통 | 차이 |
|---|---|---|---|
| **SY 라인** | 00 · 기업가치분석(SY)<br>05 · SY 평가법<br>06 · SY 가치평가 | 같은 `sy_evaluate` API<br>같은 3접근법 로직 | 00/06=단일 종목 상세<br>05=TOP10 스크리너 |
| **구 엔진 라인** | 02 · 저평가 TOP10<br>03 · 기업가치평가 | 같은 `engine.Financials` 9-모델 | 02=TOP10<br>03=단일 종목 |
| **종합** | 04 · 투자분석 | 가치평가 + 뉴스 감성 + 변동성 | 단독 — 통합 결정 페이지 |
| 독립 | 01 · DASHBOARD<br>07 · 토픽뉴스 | — | — |

### 문제점
- 사용자가 "00 기업가치분석"과 "06 SY 가치평가" 의 **차이를 인지하기 어려움**
- "02 저평가 TOP10" 과 "03 기업가치평가" 가 **같은 엔진의 다른 입력 형태**
- **두 평가법을 한 화면에 비교**할 수 있는 페이지 부재

---

## 2. 목표

1. 8개 → **5개**로 통합
2. 중복 제거 + 사용자 인지 부하 ↓
3. **단일 의사결정 페이지(04)** 신설 — SY + 다중모델 + 뉴스 감성 + 추천 통합
4. 회귀 위험 최소화 — 기존 API/엔진 그대로, **UI 라우팅·컨테이너만 재구성**

## 3. 최종 5개 페이지 구조

```
01 · DASHBOARD                       (그대로)
02 · 기업가치평가(SY)                 ← 00 + 05 + 06 통합
     ├─ 탭1: 단일 종목 상세            (구 00, 06)
     └─ 탭2: TOP10 스크리너            (구 05)
03 · 다중모델 평가                    ← 02 + 03 통합
     ├─ 탭1: 단일 종목 (9-모델)         (구 03)
     └─ 탭2: TOP10 스크리너            (구 02)
04 · 종합 분석                  ★새    ← 구 04 확장 + 비교 페이지
     ├─ SY 평가법 결과
     ├─ 다중모델 평가 결과
     ├─ 양쪽 비교 분석
     ├─ 뉴스 감성 + 변동성
     └─ 단기/장기/관망 의사결정
05 · 토픽뉴스                        (그대로)
```

## 4. 페이지별 상세 설계

### 4.1 페이지 01 — DASHBOARD (변경 없음)

- 라우트: `#/dashboard`
- 컴포넌트: `renderDashboard()`
- API: `/api/commodities`, `/api/news/market`, `/api/health`
- 변경: 없음

### 4.2 페이지 02 — 기업가치평가(SY)

- 라우트: `#/sy` (기본 탭=단일종목)
- 라우트(레거시 호환): `#/sy-analysis`, `#/sy-detail`, `#/sy-screener` → `#/sy` 로 redirect
- 컴포넌트: 새 컨테이너 `renderSy(root, params)`
  - 탭 UI 헤더
  - 활성 탭에 따라 기존 컴포넌트 호출
    - 탭1: `renderSyAnalysis()` 또는 `renderSyDetail()` (둘 중 더 완성도 높은 거 채택)
    - 탭2: `renderSyScreener()`
- API: `/api/sy/evaluate?q=...`, `/api/sy/undervalued?n=...`
- 신규 코드: ~80 lines (탭 컨테이너만)

#### 탭 전환 UX
```
┌──────────────────────────────────────────┐
│  02 · 기업가치평가(SY)                    │
│  [● 단일 종목] [TOP10 스크리너]            │
├──────────────────────────────────────────┤
│  ... 활성 탭 내용 ...                     │
└──────────────────────────────────────────┘
```

- 탭 클릭 시 URL 쿼리에 `?tab=screener` 추가 → 새로고침에도 유지
- 검색 시 자동으로 탭1 (단일)로 전환

### 4.3 페이지 03 — 다중모델 평가

- 라우트: `#/multi` (기본 탭=단일종목)
- 라우트(레거시): `#/search`, `#/undervalued` → `#/multi`
- 컴포넌트: 새 컨테이너 `renderMultiModel(root, params)`
  - 탭1: `renderSearch()` 단일 종목 9-모델
  - 탭2: `renderUndervalued()` TOP10
- API: `/api/valuation?q=...`, `/api/undervalued?n=...`
- 신규 코드: ~60 lines

### 4.4 페이지 04 — 종합 분석 ★ 신규

- 라우트: `#/analysis`
- 라우트(레거시): `#/recommend` → `#/analysis`
- 컴포넌트: 새 컴포넌트 `renderAnalysis(root, params)`
  - **2-3 API 병렬 호출**:
    - `/api/sy/evaluate?q=...` (SY 평가법)
    - `/api/valuation?q=...` (다중모델)
    - `/api/recommend?q=...` (기존 04 — 뉴스 감성 + 변동성 + 추천)
  - 한 페이지에 모두 표시
- 신규 코드: ~250 lines (가장 큰 작업)
